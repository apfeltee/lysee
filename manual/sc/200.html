2. 安装和运行
2.1. 安装

首先请检查并下载最新的安装程序，然后在安装前请仔细阅读安装程序附带的安装说明：

操作系统 说明
Windows 运行下载的安装程序即可，默认安装在 c:\lysee 目录
Linux/Unix 下载源代码，解下缩，在源代码目录输入命令“make;sudo make install”进行编译和安装

2.3. 配置

安装完毕后，请将 Lysee 的安装目录添加到系统或当前用户的搜索路径中，或为其建立调用别名：

操作系统 方法
Windows 打开“我的电脑/属性/高级/环境变量/系统变量”将 lysee 安装目录追加到 PATH 变量中即可
Linux/Unix 将命令“alias lysee='/usr/local/lysee/lysee'”追加到 ~/.bashrc 文件中

修改“lysee.config”文件调整 Lysee 内核的配置：
参数名称 说明 默认值
lse_tmpath 指定临时文件夹和上传文件的临时存放处 ${LYSEE}/temp
lse_search 指定模块搜索路径 ${LYSEE}/modules
lse_create_log 是否创建并打开日志文件 false

修改“lysee.config”文件可以使用以下预定义常量：
常量名称 常量说明 常量值
${confile}
 内核配置文件名 “lysee.config”文件完整路径名
${kernel}
 内核文件名 内核文件完整路径名
${kndir}
 内核文件所在目录 内核文件所在目录完整名称
${knpath}
 内核文件所在文件夹 内核文件所在目录完整路径名，含路径分隔符
${home}
 当前用户根目录 当前用户根目录完整路径名，含路径分隔符
${program}
 装载 Lysee 内核运行的程序 程序文件的完整路径名
${tmpath}
 临时文件夹 通过“lysee.config”文件设置，含路径分隔符
${search}
 模块搜索路径 通过“lysee.config”文件设置
${keywords}
 关键子列表 关键子列表
${mimefile} MIME文件名 安装目录下"mime.config"文件完整路径名
2.4. 测试运行

打开一个终端窗口，键入“lysee”并回车，lysee 将打印一个“>>> ”提示符等待你输入后续命令，键入“/q”退出：

D:\www.lysee.net>lysee
Lysee 0.1.6.387 - Copyright (C) 2003-2009 LiBUDI - www.lysee.net

           :: /C=CANCEL /F=FILE /Q=QUIT /R=RESTART ::

>>> /q
D:\www.lysee.net>

未出现以上画面说明 lysee 安装目录不在当前用户的搜索路径中，这时必须使用绝对路径名：

D:\www.lysee.net>c:\lysee\lysee
Lysee 0.1.6.387 - Copyright (C) 2003-2009 LiBUDI - www.lysee.net

           :: /C=CANCEL /F=FILE /Q=QUIT /R=RESTART ::

>>> /q
D:\www.lysee.net>

2.5. Hello World

第一个程序我们从“Hello World”开始，请打开一个终端窗口启动 lysee 并按示例输入“= "hello world";”：

D:\www.lysee.net>lysee
Lysee 0.1.6.387 - Copyright (C) 2003-2009 LiBUDI - www.lysee.net

           :: /C=CANCEL /F=FILE /Q=QUIT /R=RESTART ::

>>> = "Hello World";
Hello World
>>>

第一个 Lysee 程序“Hello World”做完了，简单吗？

[说明]   1. 符号“>>> ”提示操作员输入的 Lysee 代码或命令。
    2. “= "Hello World";”表示打印字符串“Hello World”，操作符“=”用于打印跟在其后的内容。
    3. Lysee 语句用分号进行分隔，交互模式下必须在最后一句代码后输入分号表示结束。
2.6. 执行程序文件

我们将代码“= "hello world";”存入文件“hw.ls”，并用 Lysee 直接在命令行执行此文件：

D:\www.lysee.net>echo ="hello world"; > hw.ls
D:\www.lysee.net>lysee hw.ls
Hello World
D:\www.lysee.net>

[说明]   1. echo 命令将其后的参数通过转向符“>”存入文件“hw.ls”。
    2. Lysee 执行完文件后自动退出。
    3. Lysee 程序文件的后缀名为“.ls”。
    4. 应将复杂的操作保存到程序文件中以方便调试和重复使用。
2.6. 命令行参数

Lysee 的命令行参数比较丰富，通过命令行“lysee --help”可以将它们打印出来：

D:\www.lysee.net>lysee --help
Lysee 0.1.6.387 - interactive LYSEE script interpreter

Usage: lysee [OPTION]... [FILE [ARGS]...]

Option:
  -v, --version           display the version of lysee and exit.
  -h, --help              print this help and exit.
  -s, --search=PATH       set module search path.
  -S, --script=SCRIPT     execute script.
  -w, --wait=SECONDS      wait for some seconds before going on.
  -t, --times=COUNT       execute specified times.
  -p, --pause             pause after execute script file.

File:
  .ls                     execute this file directly.
  .lsp, .html, .htm, .*   execute as a LSP file (Lysee Script Page).

Args:
  *                       arguments for file execution.

D:\www.lysee.net>

选项（Option）：

选项 附加参数 说明
-v --version 无 打印 Lysee 的版本信息后退出
-h --help 无 打印帮助信息后退出
-s --search 用分号隔开的搜索路径列表 为 Lysee 附加模块搜索路径
-w --wait 秒数 执行前等待指定的秒数
-t --times 次数 反复执行程序文件指定的次数
-p --pause 无 执行完程序文件后暂停

文件（File）：

文件后缀 文件类型 说明
.ls Lysee 程序或模块文件 由纯 Lysee 代码构成
.lsp LSP（Lysee Script Page）文件 嵌入 Lysee 代码的 HTML 文件，用于动态生成 web 页面
.htm HTML 文件 等同于 LSP 文件
.html HTML 文件 等同于 LSP 文件
* 其它后缀的文本文件 Lysee 解释器将其认为是 LSP 文件

参数（Args）：

Lysee 执行文件或代码（-S 选项）时，参数部分传递进内核保存在字符串列表 __args__ 中，程序代码可以直接访问到。

命令行示例：

命令行 说明
lysee backup.ls "skip=.tmp,.bak,....,.dcu,.ppu" 备份 lysee 工程目录，忽略指定的文件
lysee -w 3600 -t 9999 spider.ls 不间断，每1小时执行一次爬行机器人
lysee logout.lsp user=mack 注销用户 mack 的登陆信息
